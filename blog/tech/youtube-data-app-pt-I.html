Building a Data App Pt. I

Fecha: 2021-12-22

Autor: gmarr

Youtube is the biggest video platform in the world. This app aims to analyze the performance of a channel and also try to answer some questions such as: What are the top videos of the channel? Which videos have the highest engagement? and what's the conversation around them?

I'm going to use different technologies to build this app. Here's a list of some of them:

  • FastAPI
  • TailwindCSS
  • The Youtube API
  • ChartJS
  • HTML/JS/CSS
  • Python/Pandas
  • Docker
  • Azure VM

1. Getting the Data

The most important part of this app is the data about the channel and the videos. In order to get this data, I'm going to rely heavily on the Youtube API. This API is super robust and gives access to tons of interesting data. Here's the code I used and a brief explanation about how I used it.

1.1 API_KEY

To get access to the Youtube API you need an api_key. Here's an article from Hotspot about how to get it.

In summary, you need to have a Google account, then you must go to the Google API explorer, activate the Youtube service, and generate a key.

Once you have it, we can continue with the process.

1.2 Python Virtual Environment

Since I'm going to deploy this app on a Virtual Machine I'm going to generate a Virtual Environment with Python. This will help me to keep all my key files in a single place and keep all my python packages organized.

To create this environment I will use the next command on my terminal. (This command only works on Unix-based systems, such as MACOS or Ubuntu)

python3 -m venv env

1.3 requirements.txt

Now we´re gonna create a file called requirements.txt. This file will include the name of all the python packages that we are going to use on this project. Then we will install all those packages in our virtual environment.

touch requirements.txt

1.6 .env

The .env file is used to store all the environmental variables for our project. In our case, we're only gonna use one. The youtube_key. There are many security benefits to using this kind of file. For example, by using it the secret keys are not exposed to the client.

#.env
youtube_key = <Your Youtube API Key>

3. Adding TailwindCSS

npm init -y

Now we're going to add Tailwind to the project. To do it first we need to install the necessary dependencies inside our templates folder. We do it using the next commands:

npm install -D tailwindcss
npx tailwindcss init

This will generate a couple of folders. Open the one called tailwind.config.js and add the next snippet of code:

//tailwind.config.js

module.exports = {
  content: ["./*.html"],
  theme: {
    extend: {},
  },
  plugins: [],
}

Then open the file called input.css and add the next snippet of code:

/* input.css */

@tailwind base;
@tailwind components;
@tailwind utilities;

Finally, run the next command on the terminal.

npx tailwindcss -i ./input.css -o ./static/output.css --watch

This command will start a node server that will generate the style sheet for our project. All the styles will be saved on the static folder.
